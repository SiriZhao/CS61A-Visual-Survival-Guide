Environments — 核心概念

Environment（环境） = 一个 帧（frame） 序列。每个帧是 变量名 → 值 的映射。

• 函数被调用 → 创建新帧，参数绑定为实参值。
• 函数返回 → 该帧从环境中移除。
• 名字查找规则：在当前环境中，从最近的帧开始向外查找，返回第一个匹配的绑定。 （A name evaluates to the value bound in the earliest frame where that name is found.）

Name Rebinding — Question 1

f = min        # 1️⃣  f 绑定到 min 函数
f = max        # 2️⃣  f 改为绑定到 max 函数
g, h = min, max  # g → min，h → max
max = g        # 3️⃣  名字 max 现在绑定到 min 函数！

print(max(f(2, g(h(1, 5), 3)), 4))
逐步求值：
h(1, 5) → max(1,5) = 5
g(5, 3) → min(5,3) = 3
f(2, 3) → max(2,3) = 3（此时 f 仍是原始 max）
max(3, 4) → min(3,4) = 3（max 已被重绑为 min！）
Output: 3

Fibonacci Sequence

# 版本 1：从 (0, 1) 开始，k 从 1 计数
def fib(n):
    """Compute the nth Fibonacci number, for n >= 1."""
    pred, curr = 0, 1   # 第 0 项, 第 1 项
    k = 1               # curr 是第 k 项
    while k < n:
        pred, curr = curr, pred + curr
        k = k + 1
    return curr

print(fib(10))   # → 55

# 版本 2：从 (1, 0) 开始，k 从 0 计数
def fib_(n):
    """Compute the nth Fibonacci number, for n >= 1."""
    pred, curr = 1, 0   # 第 1 项, 第 0 项
    k = 0               # curr 是第 k 项
    while k < n:
        pred, curr = curr, pred + curr
        k = k + 1
    return curr

print(fib_(10))  # → 55
版本1「少算一步」但「起点更靠前」；版本2「起点靠后」但「多算一步」，两者恰好抵消，都在 curr 到达 55 时停下。

Control — if_ 函数的陷阱

from math import sqrt

# ✅ 正确做法：用 if 语句
def real_sqrt(x):
    if x >= 0:
        return sqrt(x)
    else:
        return 0

print(real_sqrt(-1))   # → 0
print(real_sqrt(4))    # → 2.0

# ❌ 有问题的做法：用函数实现条件
def if_(condition, true_result, false_result):
    if condition:
        return true_result
    else:
        return false_result

def real_sqrt_bad(x):
    return if_(x >= 0, sqrt(x), 0)   # ← sqrt(-1) 会在传参时被求值！

try:
    real_sqrt_bad(-1)   # 触发 ValueError: math domain error
except Exception as e:
    print('Error:', e)
调用表达式在调用前会先求值所有参数，无法跳过。
因此把条件分支封装成函数时，无论条件真假，两个分支都会被计算。
→ 需要短路求值时，只能用 if/else 语句，不能用函数。

Logical Operators — 逻辑运算符

表达式求值规则短路？
A and BA 为假 → 返回 A；否则返回 B✅ A 为假时跳过 B
A or BA 为真 → 返回 A；否则返回 B✅ A 为真时跳过 B
not AA 为真 → False；A 为假 → True—
def has_big_sqrt(x):
    """Return whether x has a real square root > 10."""
    return x > 0 and sqrt(x) > 10
    # 若 x <= 0，短路，不会调用 sqrt(x)

print(has_big_sqrt(1))       # → False
print(has_big_sqrt(1000))    # → True
print(has_big_sqrt(-1000))   # → False （短路，不求 sqrt）

def reasonable(n):
    """Return whether n is a reasonable number."""
    return n == 0 or 1 / n != 0
    # 若 n == 0，短路，避免除零错误

print(reasonable(0))     # → True （短路）
print(reasonable(100))   # → True

Generalization — 泛化：Area Functions

from math import pi, sqrt

# ── 重复的实现 ──
def area_square(r):  return r * r
def area_circle(r):  return pi * r * r
def area_hexagon(r): return 3 * sqrt(3) / 2 * r * r

# ── 泛化：提取公共结构 ──
def area(r, shape_constant):
    assert r > 0, 'A length must be positive!'   # 断言语句
    return shape_constant * r * r

def area_square(r):   return area(r, 1)
def area_circle(r):   return area(r, pi)
def area_hexagon(r):  return area(r, 3 * sqrt(3) / 2)
assert <expr>, <message>：若 expr 为假则抛出 AssertionError，显示 message。
泛化的核心：把变化的部分（shape_constant）参数化，把不变的结构提取成函数。

summation() — 函数作为参数

def identity(k):  return k
def cube(k):      return pow(k, 3)

def summation(n, term):
    """Return term(1) + term(2) + ... + term(n)."""
    total, k = 0, 1
    while k <= n:
        total, k = total + term(k), k + 1
    return total

print(summation(5, identity))  # 1+2+3+4+5  → 15
print(summation(5, cube))      # 1+8+27+64+125 → 225
term 是一个函数参数——这使得 summation 成为高阶函数。
通过传入不同的 term，同一个 summation 可以计算任意序列的和，不需要重写循环逻辑。

make_adder — 函数作为返回值 & 闭包

def make_adder(n):
    """Return a function that takes K and returns N + K."""
    def adder(k):
        return n + k       # 内层函数访问外层函数的参数 n
    return adder           # 返回函数对象本身

add_three = make_adder(3)
print(add_three(4))          # → 7
print(make_adder(5)(10))     # → 15
Closure（闭包）：adder 记住了定义时所在环境中 n 的值。
即使 make_adder 的调用帧已被移除，n 仍通过闭包保持可访问。 
def apply_twice(f, x):
    """Return f(f(x))."""
    return f(f(x))

def square(x): return x * x

print(apply_twice(square, 3))   # square(square(3)) = square(9) = 81

Environment Diagrams — 完整指南

一、核心概念速查

环境（Environment）：一系列帧，通过父链接连接，形成作用域链。全局帧位于最底层。
帧（Frame）：变量名 → 值 的映射字典。
绑定（Binding）：帧中的 x → 3 记录。
函数对象（Closure）：= 代码（参数+函数体）+ 定义时的词法环境指针。

二、绘制步骤（逐步法）

步骤操作
1画全局帧，写入所有顶层绑定（函数名、变量）。
2遇到函数调用：先求值所有参数表达式。
3为被调用函数创建新帧，写入参数名 = 实参值。
4新帧的父链接指向函数定义时的环境（非调用时！）。
5在新帧中执行函数体。
6函数返回 → 移除调用帧，返回值传给调用者。

示例 1：make_adder 闭包

def make_adder(n):
    def adder(k):
        return n + k
    return adder

add_three = make_adder(3)   # add_three 是闭包，捕获 n=3
result    = add_three(4)    # → 7
1 Global 帧：make_adder → <func>，add_three → ?
2 调用 make_adder(3)：新帧 n=3，父→Global
3 定义 adder：函数对象带箭头→ 当前帧（含 n=3）
4 make_adder 返回闭包 → Global 中 add_three = <closure adder, env{n=3}>
5 调用 add_three(4)：新帧 k=4，父→闭包环境（含 n=3）
6 计算 n+k = 3+4 = 7，返回 7。
Output: 7

示例 2：apply_twice 调用序列

apply_twice(square, 2)
→ 新帧：f=square, x=2，父→Global
→ 内层 f(x) = square(2) → 新帧 x=2 → 返回 4
→ 外层 f(4) = square(4) → 新帧 x=4 → 返回 16
→ apply_twice 返回 16
Output: 16
易错点：
1. 忘记先求值参数再创建帧。
2. 把函数的「定义环境」误认为「调用环境」——闭包捕获的是定义时的词法环境。
3. 忘记函数返回后移除调用帧（但被闭包引用的帧会长期存在）。

Function Composition — 函数组合

def square(x):  return x * x
def triple(x):  return 3 * x

def compose1(f, g):
    """Return h such that h(x) = f(g(x))."""
    def h(x):
        return f(g(x))
    return h

squiple = compose1(square, triple)   # squiple(x) = (3x)² = 9x²
tripare = compose1(triple, square)   # tripare(x) = 3(x²) = 3x²

print(squiple(5))   # 9 * 25 = 225
print(tripare(5))   # 3 * 25 = 75

squadder = compose1(square, make_adder(2))   # squadder(x) = (x+2)²
print(squadder(3))   # (3+2)² = 25

Lambda Expressions — 匿名函数

square = lambda x: x * x       # 等价于 def square(x): return x * x

print(square(5))                # → 25
print((lambda x: x * x)(3))    # → 9  （直接调用）
Lambda 是单表达式的匿名函数，语法：lambda <params>: <expression>。
不能包含语句（如 if、while、return）。
Lambda 的父帧是定义时的当前帧（同样形成闭包）。

Lambda 闭包示例

a = 1
def f(x):
    a = 2
    return lambda y: a * g(y)   # 捕获局部 a=2，不是全局 a=1

# g 作为参数传入
f(lambda y: a + y)(a)
# g = lambda y: a + y，这里 a=1（全局），所以 g(y) = 1 + y
# lambda y: a * g(y) 中 a=2（f 内局部），g(1) = 1+1=2
# 结果：2 * 2 = 4

Currying — 柯里化

def curry2(f):
    """Convert a two-argument function to curried form."""
    def g(x):
        def h(y):
            return f(x, y)
        return h
    return g

from operator import add
curried_add = curry2(add)
add_three   = curried_add(3)   # 固定第一个参数为 3
print(add_three(2))            # → 5

# 用 lambda 简化 curry2：
curry2 = lambda f: lambda x: lambda y: f(x, y)

print(curry2(add)(3)(2))       # → 5
柯里化：将多参数函数转换为一系列单参数函数的过程。
f(x, y) → g(x)(y)，允许逐步传入参数，形成更灵活的函数组合。

Return Statements — 返回语句

def end(n, d):
    """打印 n 末位数字，直到遇到 d 为止。"""
    while n > 0:
        last, n = n % 10, n // 10
        print(last)
        if last == d:
            return None   # 遇到 d 立即返回

end(12345, 3)
# 输出：5  4  3
Return 语句用于完成对调用表达式的求值。一旦执行 return，函数立即结束，剩余代码不再执行。

search() & inverse() — 高阶函数综合示例

def search(f):
    """Find the smallest non-negative integer x where f(x) is truthy."""
    x = 0
    while True:
        if f(x):
            return x
        x += 1

def square(x): return x * x

def positive(x):
    return max(0, square(x) - 100)   # x > 10 时为正

print(search(positive))   # → 11（11² - 100 = 21 > 0，为真）

def inverse(f):
    """Return g such that g(f(x)) returns x (integer approximation)."""
    return lambda y: search(lambda x: f(x) == y)

sqrt = inverse(square)
print(sqrt(256))    # → 16
sqrt(256)
→ search(lambda x: square(x) == 256)
→ 逐个测试 x=0,1,2,...,16
→ square(16) = 256 == 256 → True → 返回 16

Errors & Tracebacks — 错误与追踪

# 无限递归：f 和 g 互相调用，没有终止条件
def f(x): return g(x - 1)
def g(y): return f(y + 1)

try:
    f(3)
except RecursionError as e:
    print('RecursionError:', e)
读 Traceback 的方法：
Python 显示 "most recent call last"，从下往上读。
最底部 = 最近的调用位置（错误发生处）；往上 = 调用链的早期调用。

常见错误类型：
• RecursionError：无限递归，超过最大递归深度。
• NameError：引用了未定义的名字。
• TypeError：参数数量或类型不对。

horse() — 最难的命名混淆题

def horse(mask):
    horse = mask              # ① 局部 horse = 传入的 lambda
    def mask(horse):          # ② 局部 mask（覆盖了参数！）

    return horse(mask)        # ③ 用局部 horse 调用局部 mask

Implementing remove()

def remove(n, digit):
    """Return n with all occurrences of digit removed.
    >>> remove(231, 3)   → 21
    >>> remove(23123, 3) → 212
    >>> remove(12345, 0) → 12345
    >>> remove(0, 0)     → 0
    """
    kept, digits = 0, 0
    while n > 0:
        n, last = n // 10, n % 10
        if last != digit:
            kept = last * (10 ** digits) + kept
            digits += 1
    return kept
逐位从低位到高位扫描，保留不等于 digit 的位，按位权重重新拼装数字。
digits 记录已保留的位数，用于计算每一位的位值（10 ** digits）。

Decorators — 装饰器

def trace1(fn):
    """Return a traced version of fn (single-argument)."""
    def traced(x):
        print('Calling', fn.__name__, 'on argument', x)
        return fn(x)
    return traced

# 使用 @ 语法糖（等价于 square = trace1(square)）
@trace1
def square(x):
    return x * x

@trace1
def sum_squares_up_to(n):
    k, total = 1, 0
    while k <= n:
        total += square(k)
        k += 1
    return total

print(sum_squares_up_to(3))
Calling sum_squares_up_to on argument 3
Calling square on argument 1
Calling square on argument 2
Calling square on argument 3
14
@trace1 是语法糖，等价于在定义后立即执行 square = trace1(square)。
装饰器本质上是一个高阶函数：接受函数，返回增强版函数。
